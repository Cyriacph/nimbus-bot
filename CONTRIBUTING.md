
# Contributing to nimbus-bot

Thank you for considering contributing to nimbus-bot! We welcome bug reports, feature requests, and code contributions.


## How to Contribute

1. **Check for open issues** in the GitHub Issues tab. If you find an issue you want to work on, comment to ask to be assigned or discuss your approach.
2. **Discuss new ideas** by opening an issue or starting a discussion before submitting code.
3. **Get assigned** to an issue before starting work, to avoid duplicate efforts.
4. **Fork the repository** and create your branch from `main`. Name your branch after the issue (e.g., `fix/typo-in-readme` or `feature/add-dice-command`).
5. **Make focused commits** with clear messages.
6. **Open a pull request** referencing the issue number (e.g., `Closes #12`). Describe what you changed and why.

## Issue and Pull Request Protocol

- **Check for unassigned issues** and ask to be assigned before starting work.
- **Discuss your approach** in the issue comments if unsure.
- **Branch naming:** Use a branch name that reflects the issue (e.g., `bugfix/issue-10`, `feature/initiative-tracker`).
- **Pull requests:** Reference the issue number in your PR (e.g., `Closes #10`).
- **Keep PRs focused**—avoid mixing unrelated changes.
- **Be responsive** to feedback and requested changes.

## Community Standards
- Be respectful and constructive in all communications.

## Getting Help
If you have questions, open an issue or start a discussion.
